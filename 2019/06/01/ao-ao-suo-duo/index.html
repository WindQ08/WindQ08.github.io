一、信息收集

　　1.可以看见，原网站下面有一个客服电话，这就是一个信息
客服电话：17702979181
百度这个电话可以查得：这是一个西安的电信号码，相关信息基本都与房地产相关，机主姓名：黄经稳

　　2.电话下面有网站所属，即“西安爱家屋”
　　3.whois唯一有用的信息是本ip上只有这一个域名。
　　4.在百度下搜索西安爱家屋，发现有两个，第一个是橱柜，第二个是房地产。房地产相关的应该才是我们的目标。
房地产相关的搜索结果来源是天眼查，注册法人是黄经稳，监事是白晓雷。50w的小公司，2018年4月21日成立。
黄经稳，邮箱1637478949@qq.com，第二个电话13008830882（广东深圳），登记网址是www.ifamilyhouse.com（打开是nginx的欢迎页面），ip是118.24.234.251（上海市联通），47.74.248.20的ip地址是香港，阿里云服务器。（心凉一半，阿里云的服务器咋个整）
　　5.用nmap扫了一下所有端口（nmap -p 1-65535 47.74.248.20）。
22（ssh），80（http），3000（ppp），3306（mysql），11369（不知道干啥用的）端口是开放的。
nmap执行命令nmap -O 47.74.248.20，扫描结果中端口方面同上，其他结果如下： 

Device type: general purpose           
Running: Linux 4.X              
OS CPE: cpe:/o:linux:linux_kernel:4.4           
OS details: Linux 4.4

　　6.子域名
御剑扫描，只有两个，一个是主页面：/index.php（可知是php）
另一个是：/robots.txt（没什么价值）
　　7.还想收集信息可能就得靠社工了，如果需求什么关键信息再考虑。

二、尝试渗透

　　1.第一眼就看见mysql远程服务的3306是开着的，尝试用命令行的简单mysql语句登录，如下： 

C:\Users\HP>mysql -h 47.74.248.20 -P 3306 -u root -p root
Enter password: ****
ERROR 1045 (28000): Access denied for user 'root'@'117.32.216.11' (using password: YES)

　　说明的确连接上了，只是密码不对劲。如此尝试了很多次，每次都是密码不对，但并未禁止登录或者封ip，于是想到用msf里的mysql_login模块去破解它。
但每次破解都是只尝试了三次就终止了，提示如下：Unable to Connect: unknown charset number: 224
查了一下，应该是目标的mysql用了utf-8mb4字符集，导致无法连接。更改目标的设置是不太现实的，访问不到.ini文件，于是想是否能修改工具脚本里的字符集设置？还在研究学习中，暂时无成果。
（如果不成功，第二个想法是尝试自己猜解口令，看他两个网站的专业程度，应该是本人管理，也可以尝试从另一个网站入手。还没尝试另一个是不是也用了utf-8mb4的字符集，如果可以破解的话，说不定两个口令是同一个，其他的设置也是相同或者类似的）
　　47.74.248.20的mysql3306爆破基本失败。
　　2.第二眼看见的是提供ssh服务的22端口，尝试一下爆破它。
3.ppp是啥服务？
4.能否查出中间件的版本？如果能的话看看对应的有什么版本漏洞没有。
